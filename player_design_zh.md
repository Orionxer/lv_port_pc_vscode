# 极简音乐播放器设计说明

## 概述

本项目实现了一个用于 `320x240` LVGL 横屏显示的极简音乐播放器。

界面分为左右两个区域：

- 左侧区域：可滑动歌曲列表和音量入口按钮。
- 右侧区域：当前歌曲信息、播放进度控制、时间显示和播放控制按钮。

实现代码位于 `src/player_ui/`，通过 `src/player_ui/player_ui.cmake` 接入构建系统。

## 屏幕与布局

- 屏幕尺寸：`320x240`。
- 左侧区域宽度：`128`。
- 右侧区域宽度：`192`。
- 整体采用深色、克制的 Apple Music 风格。
- 不添加需求之外的装饰元素。

## 左侧歌曲列表

左侧区域上方是歌曲列表，下方是固定音量按钮。

- 歌曲数量：`8`。
- 每一行只显示歌曲名称。
- 左侧列表不显示歌手名称。
- 单行尺寸：`96x34`。
- 行间距：`38px`。
- 歌曲名字体：`lv_font_montserrat_16`。
- 歌曲名使用 `LV_ALIGN_LEFT_MID` 对齐。
- 歌曲名高度使用 `LV_SIZE_CONTENT`，让文字对象自身参与垂直居中，避免文字在高 label 内偏上显示。
- 当前选中歌曲使用更亮的背景色。
- 歌曲列表支持纵向滑动，隐藏滚动条。

从右侧上一首/下一首按钮切歌时，左侧列表会自动滚动，让当前选中歌曲进入可见范围：

```c
lv_obj_scroll_to_view(g_player_ui.song_buttons[g_player_ui.current_song], LV_ANIM_ON);
```

## 右侧播放器区域

右侧区域显示：

- 当前歌曲名称。
- 当前歌手名称。
- 播放进度滑动条。
- 已播放时间。
- 剩余时间。
- 上一首、播放/暂停、下一首按钮。

歌曲数据来自本地 `g_songs` 数组：

- `title`：歌曲名称。
- `artist`：歌手名称。
- `duration_sec`：歌曲时长，单位为秒。

切换当前歌曲时，通过 `update_song_view()` 统一刷新右侧信息和左侧选中态。

## 播放进度条

播放进度使用 `lv_slider`，而不是 `lv_bar`，这样可以支持触控拖动。

- 尺寸：`148x13`。
- 隐藏 knob 圆点。
- 滑动条加粗，方便触控。
- 拖动时实时更新已播放时间和剩余时间。
- 松开后提交新的播放进度到 `progress_sec`。
- 用户拖动期间，定时器不会抢占进度更新。

关键事件：

- `LV_EVENT_PRESSED`：进入拖动状态。
- `LV_EVENT_VALUE_CHANGED`：预览时间显示。
- `LV_EVENT_RELEASED` / `LV_EVENT_PRESS_LOST`：提交新的播放进度。

## 播放控制按钮

右侧底部有三个白色圆形按钮：

- 上一首。
- 播放/暂停。
- 下一首。

按钮图标使用 LVGL 内置 symbol 字体。

交互行为：

- 播放/暂停按钮切换 `playing` 状态。
- 上一首从第一首继续切换时，会循环到最后一首。
- 下一首从最后一首继续切换时，会循环到第一首。
- 上一首/下一首会将播放进度重置为 `0`。
- 上一首/下一首会刷新右侧歌曲信息和左侧选中高亮。
- 当选中歌曲超出左侧当前可视范围时，列表会自动滑动到可见位置。

## 音量入口按钮

左下角音量控制按钮是一个固定的白色圆形按钮。

- 尺寸：`42x42`。
- 图标：`LV_SYMBOL_VOLUME_MAX`。
- 点击后打开音量弹窗。

该按钮固定在左侧底部，不跟随歌曲列表滚动。

## 音量弹窗

点击左下角音量按钮后显示音量弹窗。

弹窗行为：

- 使用全屏半透明遮罩。
- 弹窗居中显示。
- 点击弹窗外部区域关闭弹窗。
- 点击弹窗内部不会关闭弹窗。

弹窗内容：

- 当前音量百分比。
- 加粗横向音量滑动条。
- 白色音量图标，无背景。

弹窗尺寸：`190x132`。

## 音量滑动条

音量控制使用 `lv_slider`。

- 范围：`0-100`。
- 尺寸：`150x18`。
- 隐藏 knob 圆点。
- 滑动条加粗，方便触控。
- 在 `LV_EVENT_VALUE_CHANGED` 中更新百分比和音量图标。

## 音量图标

音量弹窗下方显示音量图标。

当前设计：

- 无背景。
- 图标颜色为白色。
- 固定位置显示。
- 音量变化时只更新图标文本。
- 更新图标时不重新居中，也不重新布局，避免视觉跳动。

音量图标分为三档：

- `0%`：`LV_SYMBOL_MUTE`
- `1-50%`：`LV_SYMBOL_VOLUME_MID`
- `51-100%`：`LV_SYMBOL_VOLUME_MAX`

通过固定 label 位置和尺寸，减少不同 symbol 宽度变化带来的抖动。

## 构建接入

播放器模块采用独立目录组织：

- `src/player_ui/player_ui.c`
- `src/player_ui/player_ui.h`
- `src/player_ui/player_ui.cmake`

`CMakeLists.txt` 中 include `player_ui.cmake`，添加播放器 include 目录，并将 `${PLAYER_UI_SOURCES}` 加入 `MAIN_SOURCES`。

`src/main.c` 使用播放器屏幕尺寸初始化 SDL：

```c
sdl_hal_init(PLAYER_SCREEN_WIDTH, PLAYER_SCREEN_HEIGHT);
```

然后加载播放器 UI：

```c
player_ui_init();
```

## 验证方式

当前实现已使用以下命令验证：

```sh
cd build && make -j16 && cd ..
```

也使用以下命令做过短时间运行检查：

```sh
timeout 3 ./bin/main
```

运行时可能会输出 LVGL assert 或性能相关 warning，这取决于 `lv_conf.h` 配置，与播放器 UI 行为无关。
